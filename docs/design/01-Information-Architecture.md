# Information Architecture

Version: 1.0

---

# Purpose

This document defines the permanent structure of the AI Engineering Platform.

Every new feature should fit naturally into this architecture.

---

# Platform Structure

```
AI Engineering Platform

├── Dashboard
│
├── Handbook
│     ├── Introduction
│     ├── Local AI
│     ├── Prompt Engineering
│     ├── Embeddings
│     ├── Vector Databases
│     ├── Retrieval Augmented Generation (RAG)
│     ├── AI Agents
│     ├── Model Context Protocol (MCP)
│     └── Production AI
│
├── Labs
│
├── Projects
│
├── Playground
│
├── Progress
│
└── Settings
```

---

# User Journey

A new learner should naturally progress through the platform.

```
Dashboard

↓

Handbook

↓

Labs

↓

Projects

↓

Playground
```

---

# Navigation Philosophy

Navigation should always answer three questions.

- Where am I?
- Where can I go?
- What should I learn next?

The user should never feel lost.

---

# Future Expansion

Future modules should plug into this structure without redesigning the platform.
