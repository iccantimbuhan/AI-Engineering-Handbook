# Typography

Version: 1.0

---

# Philosophy

Typography is the foundation of the platform.

Reading should always feel effortless.

The interface should resemble a professionally designed technical book.

---

# Font Family

Primary

Geist

Fallback

System UI

| Element | Size | Weight |
|----------|------|--------|
| Hero | 56px | 700 |
| Page Title | 40px | 700 |

| Card Title | 20px | 600 |
| Body | 16px | 400 |
| Small | 14px | 400 |
| Caption | 12px | 400 |

# Line Height

Body

1.75

Headings

1.2

Captions

1.5

---

# Text Width

Documentation pages

Maximum width:

75 characters

Long lines reduce readability.

---

# Heading Rules

Each page should contain:

One H1

Multiple H2

Optional H3

Heading hierarchy should remain semantic.

---

# Reading Experience

Body text should never compete with headings.

Paragraph spacing should improve scanning.

Lists should remain concise.

Whitespace is considered part of typography.

---

# Design Rules

- Avoid unnecessary bold text
- Prefer hierarchy over color
- Prioritize readability
- Maintain consistent spacing between headings and paragraphs
