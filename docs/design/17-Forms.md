# Forms

Version: 1.0

---

# Philosophy

Forms should feel simple, predictable, and effortless.

Users should focus on entering information rather than understanding the interface.

Every form should reduce cognitive load.

---

# Form Structure

Each form consists of:

- Label
- Optional description
- Input
- Validation message
- Optional helper text

---

# Labels

Labels always appear above the input.

Labels should be concise and descriptive.

Required fields should display an asterisk (*).

---

# Input Height

36px

Medium

52px

# Border Radius

14px

---

# States

Default

Hover

Focus

Disabled

Error

Success

---

# Validation

Validation messages should appear below the input.

Errors should explain how to fix the issue.

Avoid generic messages.

---

# Placeholder

Placeholder text should never replace labels.

Placeholders provide examples only.

---

# Design Rules

- Keep forms short.
- Group related fields.
- Use appropriate input types.
- Always support keyboard navigation.
