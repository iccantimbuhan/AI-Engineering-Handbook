# Markdown

Version: 1.0

---

# Philosophy

Reading is the primary experience of the platform.

Markdown should resemble a professionally designed technical book rather than a blog.

Content should remain the hero.

---

# Content Width

Maximum reading width

75 characters

This improves readability during long reading sessions.

---

# Headings

Every lesson should contain:

One H1

Multiple H2

Optional H3

Headings should clearly communicate hierarchy.

---

# Paragraphs

Body text

16px

Line Height

1.75

Paragraph spacing

24px

---

# Lists

Lists should remain concise.

Nested lists should maintain consistent indentation.

---

# Code Blocks

Every code block should support:

- Syntax highlighting
- Copy button
- Horizontal scrolling
- Consistent padding

---

# Images

Images should:

- Scale responsively
- Include captions when needed
- Never overflow the reading area

---

# Tables

Tables should:

- Scroll horizontally on mobile
- Maintain readable spacing
- Support zebra striping where appropriate

---

# Future Support

- Mermaid Diagrams
- Callouts
- Tabs
- Interactive examples
- Live code playgrounds

---

# Design Rules

Documentation should prioritize readability over decoration.

Whitespace is part of the reading experience.
