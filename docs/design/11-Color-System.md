# Color System

Version: 1.0

---

# Philosophy

Color supports content.

Color should never compete with content.

Neutral colors create hierarchy while a single accent color communicates interaction.

---

# Background Colors

| Token | Value |
|-------|-------|
| Background | #FAFAFA |
| Surface | #FFFFFF |
| Sidebar | #FCFCFC |
| Muted Surface | #F5F5F5 |

---

# Border Colors

| Token | Value |
|-------|-------|
| Border | Slate 200 |
| Divider | Slate 100 |

Borders should remain subtle.

---

# Text Colors

| Token | Value |
|-------|-------|
| Heading | Slate 950 |
| Body | Slate 700 |
| Muted | Slate 500 |
| Disabled | Slate 400 |

Typography should create hierarchy rather than color.

---

# Accent Colors

Primary Accent:

Indigo 600

Hover:

Indigo 700

Soft Background:

Indigo 50

Only one primary accent should exist across the platform.

---

# Status Colors

Success

Green 600

Warning

Amber 600

Error

Red 600

Information

Blue 600

Status colors should only communicate meaning.

---

# Design Rules

- Neutral first
- Accent second
- Never use multiple accent colors on one screen
- Maintain sufficient color contrast
- Avoid gradients in application UI
