# Responsive

Version: 1.0

---

# Philosophy

The platform is mobile-first while being optimized for desktop learning.

Layouts should adapt naturally without sacrificing readability.

---

# Breakpoints

Mobile

640px

Tablet

768px

Laptop

1024px

Desktop

1280px

Wide

1536px

---

# Sidebar

Desktop

Persistent sidebar.

Tablet

Collapsible sidebar.

Mobile

Drawer navigation.

---

# Header

Search should remain the primary action.

Actions may collapse into an overflow menu on smaller screens.

---

# Dashboard

Cards should stack naturally on smaller screens.

Maintain comfortable spacing.

---

# Documentation

Reading width should remain comfortable on every device.

Avoid extremely long lines.

---

# Images

Images should scale automatically.

Never overflow their container.

---

# Tables

Allow horizontal scrolling on smaller screens.

---

# Code Blocks

Support horizontal scrolling.

Never wrap code unexpectedly.

---

# Touch Targets

Minimum interactive size

44px × 44px

---

# Design Rules

Every feature should be usable on desktop, tablet, and mobile.

Responsiveness should never compromise readability.
